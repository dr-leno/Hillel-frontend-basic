Основні команди git terminal з уроку

  • git add додати змінені файли на віддалений репозиторій
  • git commit -am флаг am дозволяє додати і підписати зміни, які були внесені перед публікацією на репозиторій одночасно
  • git commit -m після додавання файлів у репозиторій дозволяє підписати зміни, які відбулися, створити коментар
  • git push origin branchName відправляє зміни з мого комп'ютера на віддалений репозиторій, origin означає те, куди ми посилаємося
  • git checkout -b branchName створює нову гілку і переключається на неї
  • git status дозволяє подивитися, які зміни внесені були в файл після попереднього збереження на віддалений репозиторій (а також показує на якій гілці ведеться робота)
  • git fetch забирає з віддаленого репозиторію зміни, яких ще немає локально і зберігає їх
  • git merge команда для злиття гілок в одну
  • git pull не тільки забирає з віддаленого репозиторію зміни, яких ще немає, але і здиває (мержить) з кодом, з яким ведеться робота
